Maximum Likelihood Method

Setting

  • The underlying distribution of data is (supposedly) known
  • However, the parameters of that distribution are unknown
  • Goal: Estimate parameters of that distribution

Example

  • A certain random variable has distribution of N(\mu, \sigma^{2}) but not known mean and variance parameters.
  • Estimate the mean and variance
  • Maximum likelihood method will estimate values of \mu and \sigma^{2} that make our observed data most likely

Maximum Likelihood Formal Setting

Data

  • For simplicity, assume the variable of interest is continuously distributed

    • Underlying discussion applies to both discrete and continuous random variables

  • Observe n data points, Y_{1}, Y_{2}, \dots, Y_{n}.

    • Define \mathbf{Y} = (Y_{1}, Y_{2}, \dots, Y_{n}).

    • The joint pdf of \mathbf{Y} is given by f(\mathbf{Y}; \theta)

      • \theta contains the parameters of this pdf and can be a vector containing multiple parameters

Problem

  • We do NOT know what \theta is.

Technique

The maximum likelihood method is to find the parameter values of pdf which fit the observed data as closely as possible.

Likelihood Function

Definition

  • The likelihood function is simply equivalent to the probability density function (pdf): L(\theta; \mathbf{Y})= f(\mathbf{Y}; \theta).
    • This is called the joint likelihood function, since it is equivalent to the joint pdf of \mathbf{Y}.

  • The maximum likelihood estimator of \theta is \hat{\theta} = \arg \max_{\theta} L(\theta; \mathbf{Y}) .

    • ML is based on the idea that \hat{\theta} should be the value of {\theta} which maximises the likelihood of observing whatever dataset we have at hand

    • In other words, we are trying to find the value of \theta which makes the observed data as likely as possible to have been observed.

Likelihood function for IID Data

When data are independently and identically distributed, the joint likelihood function is simply the product of all marginal PDFs.

  • Suppose Y_{1},\ldots, Y_{n} are IID with PDF f(y; \theta). Then: f(\mathbf{y}; \theta) = \prod_{i=1}^{n} f_{i}(y_i; \theta)

  • Note that f(\mathbf{y}; \theta) is the joint densities and f_{i}(y_i; \theta) is for marginal densities; not the same functions

  • In practice, one usually maximises the log-likelihood function, given by l(\theta; \mathbf{y}) \equiv \ln L (\theta; \mathbf{y}): \hat{\theta} = \arg \max_{\theta}l(\theta; \mathbf{y}).

  • log is a monotonic transformation, \ln L (\theta; \mathbf{y}) and L (\theta; \mathbf{y}) are maximised at the same \theta.

  • log-likelihood is usually easier to deal with in mathematical proofs in numerical optimastion

  • For simple likelihoods, usually for iid data, the ML estimator is available in closed form.

  • When not, the maximum likelihood can still be found using numerical methods. Software tools or numerical tools (e.g. Newton-Raphson iteration).

Conclusion

  • The maximum likelihood method requires the econometrician to know (or to be happy to assume to know) the true distribution underlying data.

  • Solving the first order conditions yields the likelihood estimator (assuming that the likelihood is (strictly) concave)

  • When possible, having closed-form solutions is helpful, since it saves a lot of computation time: solving first order conditions numerically can be time-consuming.
